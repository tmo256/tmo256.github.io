Alerts

Alert WAI-ARIA Authoring

Use an alert to implement notices of success or failure for a persistence action (i.e. form submit). On a full-page view, these notices should appear in a consistent position from page to page; for example, a styled box directly beneath the main page title or the page lead text. If an alert message appears within a modal window, the alert should be prepended as the first element the modal content (body), in general (though specific layouts may require a modified implementation).

I am a Red Icon Alert

Where the alert is a direct result of an action by the user, the alert should have a dismissible option (close button). In other circumstances, alerts may be used to provide help text or other notices, and these elements should not include a close button.

Note that the example below uses an i icon element to close the alert node. This demonstrates what attributes should be included in a non-button tag to expose it as a button to a screen reader. In most cases, this code would be more like <button class='close' type="button" role="button" aria-label="Close"><i class='fa fa-close' aria-hidden="true"></i></button>. (In this fragment, the i tag sets aria-hidden to "true"; the content of the button is explained by its aria-label="Close" attribute and requires no further expository text [the icon can be skipped with no loss of information].)

Accessibility

  • Use ARIA role='alert' to display a message designed to breifly attact the user's attention. This message should not interrupt the task at hand and should therefor not affect keyboard focus.
  • For an alert that should interrupt the user's task and take keyboard focus, use ARIA role='alertdialog'. (The definition of the ARIA role='dialog' is: "an application window that is designed to interrupt the current processing of an application in order to prompt the user to enter information or require a response").

  • Alerts do not need to be visually hidden if they are not being used. However, if you do not want a screen reader to pick up an alert, use the aria-hidden='true' HTML attribute and/or display: none in your CSS.

Layout

Where possible, use semantic html to describe your HTML structure. General layout includes:

Top fixed header navigation. See “Navigation” section for more details.
Bottom “sticky” footer. See the “Footer” section for more details.
Prefer the flex method of creating a sticky footer, assuming modern browser standards.

Header Navigation

Prefer a top bar navigation with the product name on the left and the UL logo as the last element to the right, along with whatever other minimal navigation elements are required between the logo and the product name. An account icon should always be present to support at least sign out functionality, as well as settings where available, and other app-specific user features.

Applications participating in the UL platform will have an icon with a mechanism to switch to the other applications on the platform; this should be in a common location on the right, next to the UL logo. This will be similar in spirit to this functionality in Office 365 or Google and will be further defined as the platform takes shape.

In responsive sites, suppress the UL logo and turn additional navigation elements into a “hamburger” style icon as the right most element at medium (and below) breakpoints.

The code fragment below details a 60px high header with elements vertically centered. The HTML semantically describes the elements included in the header by surrounding the navigation with nav tag. (Further specifications of header navigation elements (font, case, sizing, spacing, etc.) to be added at a later date.)

<header role="header" aria-label="header">

...

<nav role="navigation" aria-label="navigation">

<ul>

<li><a>...</a>

...

</ul>

</nav>

</header>

 

header

  align-items: center;

  display: flex;

  flex-direction: row;

  height: 60px;

  position: fixed;

  width: 100%;

}

Footer

Use the semantic footer tag to designate a sticky footer (“sticky” refers to the concept of anchoring the footer to the bottom of the view if the content of the page does not push the footer beyond the bottom of the screen).>

The coding example below specifies a dark gray with white text, which should also apply any links within the section. More details on anchor and text styles may be forthcoming.

In the footer, include common elements:

the UL logo.
a copyright claim for the site content.
links to help, contact, about and other informational navigational elements not required for regular application usage.

<footer role="header" aria-label="header">...</footer>

 

footer {

  background-color: #3d3d3d;

  color: #ffffff;

}

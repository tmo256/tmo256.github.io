Layout

Where possible, use semantic html to describe your HTML structure. General layout includes:

  • Top fixed header navigation. See "Navigation" section for more details.
  • Bottom "sticky" footer. See the "Footer" section for more details.
    • Prefer the flex method of creating a sticky footer, assuming modern browser standards.
  • Main content should be semantically coded with a main tag and include the role and ARIA attributes. See "Accessibility" section for more guidelines on how to use these attributes.

Header Navigation

Prefer a top bar navigation with the product name on the left and the UL logo as the last element to the right, along with whatever other minimal navigation elements are required between the logo and the product name. An account icon should always be present to support at least sign out functionality, as well as settings where available, and other app-specific user features.

Applications participating in the UL platform will have an icon with a mechanism to switch to the other applications on the platform; this should be in a common location on the right, next to the UL logo. This will be similar in spirit to this functionality in Office 365 or Google and will be further defined as the platform takes shape.

In responsive sites, suppress the UL logo and turn additional navigation elements into a "hamburger" style icon as the right most element at medium (and smaller) breakpoints.

The code fragment below details a header with elements vertically centered. The HTML semantically describes the elements included in the header by surrounding the navigation with nav tag. See the HTML Templates section and the related HTML files for more complete code.

<header role="banner">
<div class="main-logo">
  <a href="/" rel="home" title="to homepage">
      <!-- link to home: http://microformats.org/wiki/rel-home -->
      <img alt="Logo Text/Product Name" src="/assets/images/logo.svg">
  </a>
</div>

  <!-- Quick menu (skipnav) to directly jump to specific sections: http://www.jimthatcher.com/skipnav.htm -->
  <nav>
      <ul role="menubar" aria-label="Application Menu">
        <li><a href="#" role="menuitem">App Link</a></li>
        <li><a href="#" role="menuitem">App Link</a></li>
        <li><a href="#" role="menuitem">Contact Us</a></li>
      </ul>
  </nav>

  <div id="account-menu" class="relative">
    <!-- NOTE: small screen account link/menu not wired to work -->
    <a href="#" role="menuitem" aria-expanded="false" class="account menu-icon relative;" data-class-toggle="true" data-target=".submenu" data-class="open"><img alt="Your Account" src="/assets/images/user-icon-sh.svg"></a>
  </div>

  <div id="ul-section">
    <img alt="UL" src="/assets/images/ul-white.svg" id="ul-header-logo" class="hide-sm hide-md">
    <a href="#" data-class-toggle="true" data-target="header" data-class="ham-is-active" class="hamburger hamburger--slider js-hamburger show-sm-flex show-md-flex">
      <div class="hamburger-box">
        <div class="hamburger-inner"></div>
      </div>
    </a>
  </div>

</header>

Footer

Use the semantic footer tag to designate a sticky footer ("sticky" refers to the concept of anchoring the footer to the bottom of the view if the content of the page does not push the footer beyond the bottom of the screen).

The coding example below specifies a dark gray with white text, which should also apply any links within the section. More details on anchor and text styles may be forthcoming. Again, refer to the provided templates for more detailed HTML and CSS.

In the footer, include common elements:

  • the UL logo.
  • a copyright claim for the site content.
  • links to help, contact, about and other informational navigational elements not required for regular application usage.
<footer role="header" aria-label="header">...</footer>

footer {
  background-color: #3d3d3d;
  color: #ffffff;
}

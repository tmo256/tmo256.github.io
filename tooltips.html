Tooltips & Popovers

Meta information, such as help text and input examples, may be supplied to the user via tooltips and popovers. Due to responsive requirements, using popovers poses logistical difficulties, but may be used judiciously. Length of text is the primary consideration for usage, but function also plays a role.
  • Use tooltips for short textual information attached to specific inline elements, like help for field inputs. Tooltips should contain no more than 75 characters.
  • Use a popover when providing stand-alone information, such as explanatory text or extra information of indeterminate length. For example, help that applies to an entire form (as opposed to a single element) could be contained in a popover. Popovers may also be preferred to dialogs in certain circumstances, such as upsell messages for page features.

Generally, tooltips and popover elements should behave in the same way:

  • The trigger element that opens the element should also close it. That means that all triggers toggle to open and close (per guideline specifications, elements that perform no function other than that of a toggle should always be buttons, though those buttons may be styled to appear as links).
  • In most cases, prefer triggers that open the component explicitly, rather than supplementally. For instance, do not automatically open a tooltip when a user focuses on an input field; provide a separate link to open the element.
  • Because of touch requirements, trigger both components with click events rather than hover or mouseover. This will also aid accessibility and usability concerns by avoiding unintended display of information (inadvertent mouseovers).
  • Opening a new popover or tooltip does not necessarily require closing other open elements of the same type. Generally, open tooltips should probably be closed, while open popovers should remain open.
  • Popovers may also include a separate close button in the title bar.

Accessibility

Include ARIA attributes in the trigger and display elements.

  • The element that serves as the popover/tooltip container has role=tooltip.
  • The element that triggers the popover/tooltip references the popover/tooltip element with aria-describedby.

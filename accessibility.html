Accessibility

In addition to the accessibility guidelines specified throughout the document, employ recommended best practices to accommodate ADA and general accessibility. Employing these elements will not only benefit the user, it will improve SEO on your apps. The WCAG, a W3 working group, offers extensive tutorials on coding to achieve accessibility.

ARIA attributes play a significant role in achieving accessibility. ARIA (formally, WAI-ARIA) is a set of attributes added to HTML elements to communicate the role, state and property semantics to assistive technologies, via the accessibility APIs implemented in browsers.

Coding General HTML Elements

Accessibility should be implemented by conforming to semantic HTML practices, as well as using ARIA attributes/labels. See individual component and layout sections for suggested implementation of semantic HTML.

Also note that in many cases, Javascript is inherently more accessible than CSS solutions. For example, when toggling visibility of an element, CSS psuedo selectors in combination with sibling selectors may accomplish a toggle. Consider the following code, which displays a different price based on the selection of a radio button:

<style>
[value="yearly"]:checked ~ .monthly {
  display: none;
}

[value="monthly"]:checked ~ .monthly {
  display: block;
}
[value="monthly"]:checked ~ .yearly {
  display: none;
}
</style>

<input type="radio" name="timeframe" id="timeframe_yearly" value="yearly" checked="checked">
  <label label-for="timeframe_yearly">Yearly</label>
<input type="radio" name="timeframe" id="timeframe_monthly" value="monthly">
  <label label-for="timeframe_yearly">Monthly</label>

<span class="yearly">$100/year</span>
<span class="monthly">$10/month ($120/year)</span>

In this scenario, the CSS attributes do not convery to the browser any information about what text is visble, resulting in misinformation for the user. Using Javascript in combination with aria attributes for toggled elements, however, allows the developer to accomplish the task while accommodating accessibility.

Using ARIA is particularly important in forms and inputs, while "role" attributes are particularly useful in elements (like buttons) that perform crucial functions but do not semantically indicate their uses. Refer to the WCAG documentation for use in specific HTML elements and common components (such as tooltips).

"WAI-ARIA Authoring Practices" details specific instructions for all the commonly used components included in this documentation. Look for usage of these attributes:
  • role
  • aria-*
  • aria-* attributes whose values toggle depending on state

Images

Google's Lighthouse and Page Speed tools take special interest in image accessibility. Be sure to include alt text on all img tags, as its absence is detrimental to your audit results and SEO. This alt text should describe the image to provide the same level of understanding that a visual user would have. Where possible, do not use images for buttons, navigation or significant text. Minimize text in images in general; if text is present in images, use that text as the alt value.

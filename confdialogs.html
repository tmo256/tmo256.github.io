Confirmation Dialogs

Use a confirmation dialog to confirm a process or persistence that cannot be easily undone or recreated; or, to confirm data persistence where the user may not be aware that persistence is about to occur (form inputs are not visible, for example). Most commonly, use this type of dialog when deleting or removing items. The dialog should interrupt the process to the point that the action cannot complete without explicit input (e.g., button click) from the user.

Limit usage of this pattern; avoid where possible. Alternatively:

  • When deleting an item, is it possible to include an "Undo" button?
  • Instead of a confirmation dialog when navigating away from in-progress forms, is it possible to autosave input as it's added?
  • Instead of a full modal, is a popover or similar component adequate?

When creating a confirmation dialog, text should be clear and consistent.

  • The dialog header and body text should clearly convey the action and the consequences of taking said action. Emaphsize data specific to this action, such as the user name and id of an account to be removed.
  • As described in "Anchors and Buttons," label your button with a verb; ie. "Delete account" as opposed to "Account deletion."
  • Choose a verb that cannot be confused for another action. "Cancel" is a commonly used word within sites and applications, so "Cancel account" may be confusing.
  • The modal should offer two buttons to the end user, one that confirms the action and one that interrupts it. Again, this text should be clear, so avoid buttons labeled "OK" or "Cancel." Instead, use "Keep account" (stopping the initial action) and "Remove account" (confirming the action).
  • If possible, the label on the completion button should match the label on the button (or link) that prompted the dialog initially. At a minimum, use the same verb on both; if a delete confirmation is prompted by a "Delete" link from within a table listing, for instance, your completion button may include a specific such as "Delete bookmark."
  • In some (high-stakes) situations, consider putting the emphasis on the button that cancels the action rather than on the button that completes it.
  • The action's stop button should always be on the left; the complete button should always be on the right.
  • In most cases, either action should close the dialog box.
  • If the action has been completed, display a clear success message reiterating some of the specific data from the dialog.

  • On the trigger link or button, consider including some kind of indication that another step is forthcoming, such as an ellipsis.
